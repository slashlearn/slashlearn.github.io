Intro:

If we think back to the last two lessons you might have noticed something. When we declared some variables sometimes we used capitals for the type like we did with Person bobby = new Person(); or Rectangle rect1 = new Rectangle(); (Person and Rectangle are capitalized) Other times they were lower case, int width = 10; (int is lower case). This is because Java categorizes variables in two ways: either they are one of the 8 primitive types, or they are a reference to an object. Let's explore the primitive types and object references further.



8 Primitive Types:

Primitive types are the basic building blocks of a Java program. If we make an analogy to chemistry we can think of them as the individual atoms of a molecule.

Let's go a bit more in depth on the more commonly used primitive types

int:

  1. Integers are whole numbers and can be positive or negative
  2. Integers have a limited range -2147483548 to 2147483547 which can cause overflow issues. This occurs when a integer has exceeded its maximum value and "wraps" around to become the most negative value. (see picture)
  3. Use the long primitive types if you need a larger range.

booleans:

  1. booleans can hold only two values, either true of false.
  2. Will become useful later when we want to control the flow of a program. (If a boolean is true do something, if it is false do something else).
  3. You can initialize with value true or false, or with a boolean expression: an expression which will evaluate to true or false. (See second picture)



double:

  1. doubles can hold fractional/decimal values
  2. Limited range but it is so large (10^308) in most cases you do not have to worry.
  3. Be careful of rounding issues (see right picture). These occur because numbers are represented in the computer as binary 1's and 0's. In our decimal number system there are certain numbers that we can not represent exactly. For example the closest we can get to 1/3 is .3333 repeating. Similarly our computers can not represent some numbers exactly in binary.


Casting:

Casting occurs when you convert from one primitive type to another. For example, converting from a double to an int or int to double.
