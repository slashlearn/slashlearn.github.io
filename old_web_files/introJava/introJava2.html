Types:

Whether it’s a word, number, or something more complex, Java needs to be able to categorize values, and it does so by “type.”

Some examples of Types are:

  • Integer: whole number —ex: 5
  • Double: number that can hold a fractional value —ex: 5.46
  • String: sequence of characters —ex: “Hello my name is Bobby”

Types determine what operations you can use on them. For example:







You are able to multiply two integers (5 * 2).


However, notice how when we try and multiply two strings Eclipse throws out an error. This is because the string type does not allow you to carry out the multiplication operation. (Multiplying two words together doesn’t really make sense)

Assignment (=):

What happens if we want to change the value of a variable. Well we can do this by re-assigning a value to that variable. We do this with the assignment operator (=). Something important to remember is that in Java (=) does not mean a comparison for equality rather it is used to assign the value on the right hand side to the variable on the left hand side of the (=) operator.

Variable Naming Rules and Conventions:

Here are some variable naming rules. These will cause an error if you do not follow them.

  1. You can use numbers, letters, $, and underscores, but you can't start with a number
  2. No Spaces and most symbols are not allowed: % and & to name a few
  3. Java has certain reserved words like public, private, and static you can not use these as your variable names; however, you can have them in your variable names.
    For example:
    -int public = 1; This is not ok
    -int publicSchoolID = 2841; This is ok

Here are some naming conventions, these will not technically give you an error if you break them, but it is good practice to follow anyways.

  1. Variable and method names should start with lower case. (We will talk more on methods later)
  2. Class names should start with a Capital
  3. Clear but Concise
  4. Use camelCase. This is where we start variable names in lower case but capitalize everyword after the first in a variable name.
    For example:
    -bankAccount
    -johnAdams

Side Note: Why might it be good practice to follow the conventions? Well this is the "standard" among Java programmers which means if you ever look at production level code it will follow these conventions. It is good practice to get in the habit of following these conventions now, so that if you perhaps work as a software engineer in the future other people will be able to read your code.
